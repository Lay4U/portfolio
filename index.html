VIEW MORE
차선 인식 시스템을 개발하기 위하여 컴퓨터 과학과 4명이 조를 이루어서 프로젝트를 진행하였다.
카메라 왜곡을 보정하기 위해 OpenCV의 Camera Calibration을 사용하였고 Sliding Window Search를 위한 Binary Image를 만들기 위해 Filtering단계를 거쳤다 이때 필터는 Canny Filter을 사용하였다. 고정된 thresholding 값으로 필터링을 하면 차선이 인식이 안될때가 있어 Sobel Operator를 같이 사용하여 개선된 이미지를 얻었다. 차선을 인식하기 위하여 Region of Interest를 사용하였고 Masked된 이미지를 추출할 수 있었다. 또한 곡선 차선을 인식하기 위하여 Sliding Window Search Method를 이용하였다. Sliding Window로 인식한 차선 데이터를 사용자가 볼 수 있도록 원본의 이미지에 인식 결과를 표시하는 Superimposition을 적용하여서 프로젝트를 완성시켰다.
쉽게 생각했던 차선 인식은 의외로 매우 어려운 작업이였다. 많은 기하학적 지식을 요구하였고 이미지를 추출하는데 있어 최적의 조합을 구하는것이 매우 어려운 일이였다... 그래도 인터넷의 자료와 OpenCV를 기반으로 여러 조합을 찾고 개선해 나가면서 정확도를 올렸다. 많은 솔루션들이 자세하게 설명되어 있었고 이러한 솔루션들을 적용시켜 가면서 영상 처리를 위한 알고리즘을 이해하고 성능도 높이면서 차선을 인식할 수 있는 프로젝트를 성공적으로 완성 시킬 수 있었다.

PROJECT 10

알고리즘

Brute-Force, 분할 정복법과 다이나믹 프로그래밍을 이용하여 알고리즘 문제를 해결하는 프로젝트를 진행하였다.

VIEW MORE
분할 정복방법과 다이나믹 프로그래밍을 이용한 문제를 풀기 위한 프로젝트로 컴퓨터과학과 4명이 팀을 이뤄 프로젝트를 진행했다.
분할 정복 알고리즘은 문제를 풀기 위해 크기를 줄여 나가는 방식이다. 분할 정복의 기본 매커니즘은 큰 문제를 여러개로 쪼갠뒤 다시 합쳐 정답을 구하는 방식이다. 문제의 분할(Divide), 정복(Conquer), 병합(Combine)과정을 거쳐 분할 정복 알고리즘을 구현 할 수 있었다. 다이나믹 프로그래밍은 크고 어려운 문제가 있으면 그것을 먼저 잘게 나누어서 해결한 뒤에 처리하여 나중에 전체의 답을 구하는 것이다.이 과정에서 Memoization이 사용된다는 점에 분할 정복과 다르다 즉, 주어진 부분 문제의 정답을 한 번만 계산하고 저장해둔 뒤 다시 한 번 이 부분 문제를 이용할 때에는 저장해둔 정답을 바로 산출하여 이용함으로써 속도를 향상시킨다. 위의 두 알고리즘을 사용하여 여러 문제를 푸는 프로젝트를 진행 하였고 문제를 풀 때 마다 많은 고민을 갖는 시간을 가졌다. 여러가지 경우의 수를 두고 팀원들과 회의를 많이 가지면서 어떻게 문제를 풀어야 할 지에 대해서 의견을 나누면서 점진적으로 문제를 해결하였고 두 가지 알고리즘에 대한 개념을 토대로 문제를 풀 수 있는 프로젝트였다.

VIEW MORE
여러가지 자료구조를 이용해서 시간 복잡도 및 공간 복잡도를 계산하는 프로젝트를 개인 프로젝트로 진행하였다. 자료구조에는 Array, Linked List, Stack, Queue, Deque, Tree, Binary Tree, Graph, Hash Table등을 구현하였고 데이터는 user, friendship relation, word로 구성되어 있다. 각 데이터를 Linear Search, Binary Search, Hash Search, BFS, DFS, Uniform-Cost Search, Depth-limited search, Iterative Deepening Search, Adimissible Heuristic, Greedy Algorithm, A* Algorithm등을 사용하여 각 탐색의 결과값을 시각화 해서 프로젝트를 마무리 했다. 여러가지 자료구조를 수업시간때 배운 내용을 기반으로 하나씩 만들어 가는 작업이 힘들었지만 지속적으로 개선하고 개발하다보니 패턴이 눈에 보이고 손에 익어서 후반대에 갈 수록 빠른 작업이 가능해졌다. 여러 자료구조와 이를 탐색하는 방법에대한 기초적인 이론을 기반으로 실제 구현을 통해 기본을 다질 수 있는 좋은 프로젝트였다.

PROJECT 12

DeadLock Avoid

가상의 학습 OS를 컴파일 하여 OS를 수정및 컴파일하여 DeadLock을 avoid 하기 위한 Banker’s Algorithm을 구현하는 프로젝트를 진행하였다.

VIEW MORE
DeadLock을 회피 할 수 있는 Banker's Algorithm을 구현하기 위해 nachOS 환경에서 컴퓨터과학과 학생 2명이 만든 프로젝트이다.
Deadlock 발생 조건은 Mutual Exclusion, Hold and Wait, No Preemption, Circular Wait가 있다. 그리고 Deadlock의 해결 방법에는 회피 기법, 발견 기법, 회복 기법등이 있다. 그중에서 Avoid의 Banker's Algorithm을 구현하여 DeadLock을 피해 갈 수 있도록 프로젝트를 진행했다. 알고리즘은 은행에서 DeadLock에 빠질 가능성이 있는지 판단하기 위해 safe state와 unsafe state로 구분한다. 그리고 safe state를 유지 할 수 있는 만큼의 요구만을 허락해주고 unsafe state 상태의 가능성이 있는 요구는 계속 거절하는 방법이다. 자료 구조는 Available, Max, Allocation, Need로 구성되어 safe를 만족하는 조건아래 에서 프로세스에게 자원을 할당해주었다. 자바로 멀티 스레드 프로그래밍을 처음 접해봤는데 디버깅 하는 과정에서 신기한, 처음보는 장면들이 인상적이였다. 또한 학습용 OS인 nachOS를 컴파일 할 수 있어서 운영체제가 어떻게 돌아가는지, 또한 DeadLock을 회피하는 알고리즘을 구현함으로써 OS와 멀티 쓰레드에 대한 기본적인 이해와 접근이 가능한 프로젝트였다.

대학교 들어와서 첫번째 프로젝트였다. 자바를 사용해 본적이 없어서 처음에는 힘들었는데 if문 for문 배열등은 C와 비슷해서 아주 어렵지는 않았다. 진도는 배열까지 나갔었고 이 배열을 활용하여 연락처를 만드는것을 주제로해서 프로젝트가 진행되었다. 각 연락처를 보기/추가/수정/삭제 할 수 있는 기능을 구현하는데 첫 프로젝트라 그런지 애를 조금 먹었다 오류나는 부분에 대해서 브레이크 포인트를 찍고, 디버깅을 하면서 java.lang.IndexOutOfBoundsException, java.lang.NullPointerException을 해결 할 수 있었고 디버깅을 하면서 각 변수의 값이 어떻게 변화하는지 한줄 한줄 실행해 가면서 확인할 수 있었고 for문과 배열에 대한 확실한 이해를 해 나가며 자바 프로그래밍에 대한 이해도가 높아졌고 조금 더 어려운 패턴의 기능을 구현 할 수 있었다. 인터넷을 검색해 가면서 여러가지 자료구조를 보았고, 여러가지 자료 구조들과 탐색 알고리즘을 적용하면서 많은 데이터를 빠르게 처리 할 수 있도록 프로젝트를 진행했다. 첫번째 자바 프로젝트이여서 두려움이 앞섰다. 이클립스라는 툴도 익숙하지 않았는데 계속해서 사용해 나가면서 여러 기능들을 사용해 보았고 디버깅 기능을 적극 활용하면서 프로젝트를 잘 마무리 하였다.
